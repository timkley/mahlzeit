# CLAUDE.md — Calorie Tracker (Cloudflare Workers + Nuxt)

## Project Overview

A simple, single-user calorie tracker for my wife. She uploads a food photo or types a meal description, and AI estimates the calories. Built on Cloudflare Workers with Nuxt, D1 (SQLite), R2 (image storage), and Workers AI (vision model).

This is my first Cloudflare Workers project. I'm an experienced Laravel developer switching to TypeScript/Nuxt for this project.

## Tech Stack

- **Framework**: Nuxt 4 with `@nuxthub/core` module (v0.10+)
- **Database**: Cloudflare D1 (SQLite) via Drizzle ORM
- **ORM**: Drizzle ORM (`drizzle-orm` + `drizzle-kit`)
- **File Storage**: Cloudflare R2 (via NuxtHub `hub:blob`)
- **AI**: Cloudflare Workers AI — use `@cf/meta/llama-3.2-11b-vision-instruct` for image analysis
- **Auth**: Simple — either Cloudflare Access (Zero Trust) or a basic password middleware. Not a full auth system.
- **Styling**: Tailwind CSS (or UnoCSS) — keep it clean and mobile-first, this will primarily be used on a phone
- **Deployment**: Cloudflare Workers via Wrangler CLI
- **Language**: TypeScript throughout

## Prerequisites & Setup

### 1. Install Node.js (if not present)
```bash
# Use volta or nvm
curl https://get.volta.sh | bash
volta install node
```

### 2. Create the project
```bash
npm create cloudflare@latest calorie-tracker -- --framework=nuxt
cd calorie-tracker
```

### 3. Install dependencies
```bash
# NuxtHub core module
npx nuxi module add @nuxthub/core

# Drizzle ORM for D1/SQLite
npm install drizzle-orm
npm install -D drizzle-kit

# Tailwind (if not already included by Nuxt scaffold)
npx nuxi module add @nuxtjs/tailwindcss
```

### 4. Configure nuxt.config.ts
```typescript
export default defineNuxtConfig({
  modules: ['@nuxthub/core', '@nuxtjs/tailwindcss'],
  hub: {
    // D1 database (SQLite)
    db: 'sqlite',
    // R2 blob storage for food images
    blob: true,
    // AI binding for Workers AI
    ai: true,
  },
  // Needed for Cloudflare Workers compatibility
  nitro: {
    preset: 'cloudflare',
  },
})
```

### 5. Wrangler login
```bash
npx wrangler login
```

### 6. Local development
```bash
npx nuxt dev
```
D1 and R2 are simulated locally via Miniflare. Workers AI calls go to Cloudflare's remote API (requires internet + logged-in wrangler).

### 7. Deploy
```bash
# Generate migrations first
npx nuxt db generate
# Deploy (Wrangler auto-provisions D1 + R2 on first deploy)
npx wrangler deploy
```

## Database Schema (Drizzle ORM)

Define in `server/db/schema.ts`:

```typescript
import { sqliteTable, text, integer, real } from 'drizzle-orm/sqlite-core'

export const meals = sqliteTable('meals', {
  id: integer().primaryKey({ autoIncrement: true }),
  // What was eaten
  description: text().notNull(),
  // Calorie estimate (user can override AI estimate)
  calories: real().notNull(),
  protein: real(),        // grams, optional
  carbs: real(),          // grams, optional
  fat: real(),            // grams, optional
  // Image reference (R2 key, nullable for text-only entries)
  imageKey: text('image_key'),
  // AI's raw response for debugging/transparency
  aiResponse: text('ai_response'),
  // When this meal was eaten
  mealDate: text('meal_date').notNull(),  // ISO date string YYYY-MM-DD
  mealTime: text('meal_time'),             // 'breakfast' | 'lunch' | 'dinner' | 'snack'
  // Timestamps
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull().$defaultFn(() => new Date()),
})
```

After defining: run `npx nuxt db generate` then `npx nuxt db migrate` (or just start dev server).

## Workers AI Integration

### Food Image Analysis

Use `@cf/meta/llama-3.2-11b-vision-instruct` for analyzing food photos.

Key implementation details:
- Images must be sent as **base64 data URIs** (not HTTP URLs)
- Set `max_tokens: 1024` (default 256 is too short)
- Set `temperature: 0.3` for consistent estimates
- Request JSON output for easy parsing

### Prompt Template (server/utils/ai-prompt.ts)

```typescript
export function buildFoodAnalysisPrompt(description?: string): string {
  const descPart = description
    ? `The user describes this meal as: "${description}".`
    : ''

  return `You are a nutritionist assistant. Analyze this food image and estimate the nutritional content.
${descPart}

Follow these steps:
1. Identify each food item visible in the image
2. Estimate portion sizes in grams
3. Calculate calories and macros per item using standard nutritional data
4. Sum up the totals

Respond ONLY with valid JSON in this exact format, no other text:
{
  "items": [
    {
      "name": "grilled chicken breast",
      "portion_grams": 150,
      "calories": 248,
      "protein": 46,
      "carbs": 0,
      "fat": 5
    }
  ],
  "total": {
    "calories": 248,
    "protein": 46,
    "carbs": 0,
    "fat": 5
  },
  "confidence": "high"
}`
}
```

### Calling Workers AI from a server route

```typescript
// server/api/analyze/image.post.ts
export default defineEventHandler(async (event) => {
  const body = await readBody(event)
  // body.image = base64 string, body.description = optional text

  const ai = hubAI()  // NuxtHub AI binding

  const messages = [
    {
      role: 'user',
      content: [
        { type: 'image', image: body.image },  // base64 data
        { type: 'text', text: buildFoodAnalysisPrompt(body.description) },
      ],
    },
  ]

  const response = await ai.run('@cf/meta/llama-3.2-11b-vision-instruct', {
    messages,
    max_tokens: 1024,
    temperature: 0.3,
  })

  // Parse AI response and return structured data
  return response
})
```

## Image Upload Flow

1. Client: User takes/selects photo → resize client-side to max 1200px width (saves bandwidth, stays within limits)
2. Client: Convert to base64 → send to `/api/analyze/image` with optional text description
3. Server: Store original image in R2 via `hubBlob().put()`
4. Server: Send base64 to Workers AI for analysis
5. Server: Save meal entry to D1 with AI results + R2 image key
6. Client: Display result with option to manually adjust calories

Workers accept request bodies up to **100MB** (paid plan). Phone photos at 2-10MB are fine. Client-side resizing before upload is recommended.

## Important Cloudflare-Specific Notes

- **No PHP, no Node.js runtime** — this runs on V8 isolates (like a browser's JS engine)
- **No `eval()`** — blocked for security
- **128 MB memory limit** per isolate — don't buffer multiple large images simultaneously
- **Workers AI always runs remotely**, even in local dev — requires internet connection
- **D1 is SQLite** — no PostgreSQL features, no stored procedures, no LISTEN/NOTIFY
- **wrangler.jsonc is preferred** over wrangler.toml for new projects
- **NuxtHub v0.10+ auto-generates wrangler config** from nuxt.config.ts — you typically don't need a manual wrangler.jsonc
- **D1 migrations are NOT auto-applied on deploy** — always run `npx nuxt db generate` then `npx wrangler d1 migrations apply <DB_NAME>` before `npx wrangler deploy`
- **Compatibility date** should be set to today's date: `2026-03-22`

## Design Guidelines

- **Mobile-first**: Primary usage is on a phone, quick meal logging
- **Simple and fast**: The main screen shows today's calories and a big "Add Meal" button
- **German locale**: The user is German — dates in DD.MM.YYYY format, use comma as decimal separator in display
- **Dark mode**: Support both light and dark mode
- **Minimal chrome**: No unnecessary navigation, no onboarding flows, no settings pages for v1
